Analysis
🤖
ML Condition Classifiers
11 independent models
Each model is trained on NHANES data to detect one condition. Outputs a probability score per condition independently.
Anemia Thyroid Kidney Liver Sleep disorder Prediabetes Iron deficiency Perimenopause Inflammation HepatitisBC Electrolytes
→ P(condition) per disease
📊
Bayesian Update
Probability refinement
Triggered when any condition clears 40% probability. Asks the patient targeted follow-up questions. Each answer updates the probability using a likelihood ratio.
Triggered at P > 0.40 Condition-specific Qs
→ Refined P(condition)
🔬
KNN Population Matching
Lab signal inference
Finds the 50 most similar people in our 7,437-person NHANES database. Surfaces labs that are disproportionately abnormal in that group, ranked by how anomalous they are vs. the general population.
K = 50 neighbours Cosine similarity Lift ≥ 1.5× 34 labs tracked
→ Ranked lab signals + direction

Filtering & Ranking
Signal selection
Selects the top 3 conditions above threshold. Merges ML posteriors, Bayesian updates, and KNN lab signals into one structured input for the LLM.

MedGemma — LLM Synthesis
Personalised report generation
Receives the structured signal package: top-3 conditions with probabilities, Bayesian clarification Q&A, and KNN lab signals. Generates a fully personalised report — every sentence references this patient's specific data, never generic advice.
Condition insights Lab test recommendations Doctor Kit script GP questions + RAG on fatigue literature

Safety
🛡️
Safety Layer
Every LLM output passes through a safety filter before being shown to the user. Removes diagnostic language, softens alarming phrasing, and enforces hard-coded guardrails.
Hard rules (code)
No diagnosis statements · "May suggest / could indicate" framing enforced · Disclaimer on every report
Output validation
JSON schema validation · AI risk assessment matrix · NeMo guardrail integration (planned)
