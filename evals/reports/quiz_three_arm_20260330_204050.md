## Notes

- `MedGemma-only` uses the live `/api/deep-analyze` path with quiz answers only and an eval-specific prompt mode that withholds ML scores.
- `MedGemma + follow-up` uses the same eval mode but also sends human-readable Bayesian follow-up Q/A without LR values or condition labels.
- On protected preview deployments, pass `--eval-secret` so the runner can access the guarded `medgemma_only` route.
- `Hybrid` passes only the top-5 model scores to the same route.
- `healthy over-alert` counts any surfaced diagnosis on healthy profiles.
- `neighbour false positive` measures how often a condition appears on clinically adjacent profiles where it is absent.
